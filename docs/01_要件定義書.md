# 要件定義書 - 戸塚博エージェント

## 1. プロジェクト概要

### 1.1 プロジェクト名
戸塚博エージェント（Totsuka Hiroshi Agent）

### 1.2 目的
YouTubeチャンネル「令和ヨットスクール」に出演する戸塚博氏のキャラクターを忠実に再現したAIチャットエージェントを開発する。ユーザーが戸塚氏と会話しているかのような体験を提供する。

### 1.3 背景
- 「令和ヨットスクール」は戸塚氏が出演するYouTubeチャンネル
- 戸塚氏の独特な口調・思想・教育哲学をAIで再現したい
- 身内・少人数での利用を想定

---

## 2. 機能要件

### 2.1 コア機能

#### F-001: チャット会話機能
- ユーザーがテキストを入力し、戸塚氏のキャラクターで応答を返す
- リアルタイムでのストリーミング応答対応
- 会話の文脈を保持した連続的な対話が可能

#### F-002: キャラクター再現機能
- 戸塚氏の口調（語尾、言い回し、方言等）を再現
- 戸塚氏の思想・教育哲学を反映した回答を生成
- 「令和ヨットスクール」での発言内容に基づく一貫性のある応答

#### F-003: YouTube文字起こしデータ収集機能
- 「令和ヨットスクール」チャンネルの動画一覧を取得
- 各動画の字幕/文字起こしデータを自動取得
- 戸塚氏の発言部分を抽出・整理してナレッジベース化

### 2.2 サブ機能

#### F-004: 会話履歴管理
- セッション内の会話履歴を保持
- 過去の会話を参照可能（ブラウザのローカルストレージ）

#### F-005: レスポンシブUI
- PC・スマートフォンの両方で快適に利用可能

---

## 3. 非機能要件

### 3.1 性能
- 応答開始まで3秒以内（ストリーミング開始）
- 同時接続数: 5人程度（身内利用のため）

### 3.2 セキュリティ
- APIキーはサーバーサイドで管理、フロントに露出しない
- 簡易的なアクセス制限（パスワード認証 or Basic認証）

### 3.3 可用性
- 身内利用のため、99%稼働率は不要
- Vercelの無料枠で運用可能な範囲

### 3.4 保守性
- コードは読みやすく、メンテナンスしやすい構成
- 文字起こしデータの追加・更新が容易

---

## 4. 制約条件

### 4.1 技術的制約
- Claude API（Anthropic）を使用
- YouTube Data API v3 / youtube-transcript-api を使用
- 無料〜低コストで運用可能な構成

### 4.2 データに関する制約
- 学習データはYouTube動画の文字起こしから生成
- 戸塚氏の発言を忠実に再現（ガードレールなし）

### 4.3 運用制約
- 利用者は身内・少人数に限定
- 一般公開は想定しない

---

## 5. ユースケース

### UC-001: 通常の会話
1. ユーザーがチャット画面を開く
2. （初回のみ）簡易パスワードを入力
3. テキストで質問や話題を投げかける
4. 戸塚氏のキャラクターで応答が返る
5. 会話を続ける

### UC-002: 特定の話題について聞く
1. ユーザーが「教育について」「若者について」等のテーマで質問
2. 戸塚氏の思想に基づいた回答が返る
3. 深掘りの質問にも一貫性をもって応答

### UC-003: 文字起こしデータの更新
1. 管理者が新しい動画の文字起こしスクリプトを実行
2. 新しいデータがナレッジベースに追加される
3. エージェントの応答に反映される

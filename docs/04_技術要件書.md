## 2. 開発環境セットアップ

### 2.1 必要なツール
- Node.js 20.x 以上
- npm or pnpm
- Python 3.10 以上
- Git

### 2.2 環境変数

```env
# .env.local（ローカル開発用）

# Anthropic API Key
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxx

# 認証パスワード
AUTH_PASSWORD=your-secret-password

# セッションシークレット（トークン署名用）
SESSION_SECRET=random-secret-string
```

### 2.3 セットアップ手順

```bash
# 1. リポジトリクローン
git clone <repo-url>
cd totsukaagent

# 2. Node.js 依存関係インストール
npm install

# 3. 環境変数設定
cp .env.example .env.local
# .env.local を編集して API キー等を設定

# 4. 開発サーバー起動
npm run dev

# 5. (データ収集) Python環境セットアップ
cd scripts
pip install -r requirements.txt

# 6. (データ収集) 文字起こし取得
python fetch_transcripts.py

# 7. (データ収集) ナレッジベース構築
python build_knowledge.py
```

## 4. Claude API 利用設計

### 4.1 モデル選定

| モデル | 用途 | 理由 |
|--------|------|------|
| claude-sonnet-4-20250514 | チャット応答 | コスト効率と品質のバランス |

### 4.2 API呼び出しパラメータ

```typescript
{
  model: "claude-sonnet-4-20250514",
  max_tokens: 1024,
  stream: true,
  system: systemPrompt,    // キャラ定義 + ナレッジベース
  messages: conversationHistory
}
```

### 4.3 トークン管理

- システムプロンプト: ナレッジベース含めて最大50,000トークン程度を想定
- 会話履歴: 直近20ターン程度を送信（古いものから切り捨て）
- 1応答あたり最大1,024トークン

### 4.4 コスト見積もり

| 項目 | 概算 |
|------|------|
| システムプロンプト（入力） | 〜50,000 tokens/リクエスト |
| 会話履歴（入力） | 〜5,000 tokens/リクエスト |
| 応答（出力） | 〜500 tokens/リクエスト |
| 1リクエストあたりコスト | 約 $0.02〜0.03 |
| 月間利用量（1日20回×30日） | 約 $12〜18/月 |

※ 身内少人数利用なら月 $20 以下で収まる想定

## 6. ナレッジベースのサイズ制限と対策

### 6.1 問題
- Claude APIのコンテキストウィンドウに上限がある
- 全文字起こしデータが大きすぎる場合、システムプロンプトに収まらない

### 6.2 対策（段階的）

**Phase 1: 全文埋め込み（初期）**
- 文字起こしデータが少ない段階ではそのまま全文をシステムプロンプトに含める
- 目安: 50,000トークン以下なら問題なし

**Phase 2: 要約・厳選（データ増加時）**
- 重要な発言を厳選してナレッジベースを圧縮
- トピックごとに代表的な発言を選定
- build_knowledge.py で自動要約処理を追加

**Phase 3: RAG導入（将来）**
- 必要に応じてベクトル検索（RAG）を導入
- ユーザーの質問に関連する発言のみを動的に取得
- 現時点では不要（身内利用・データ量限定のため）

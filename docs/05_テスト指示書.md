# テスト指示書 - 戸塚博エージェント

## 1. テスト方針

本プロジェクトは身内・少人数利用のため、自動テストは最小限とし、手動テストを中心に品質を担保する。

---

## 2. テストカテゴリ

### 2.1 データ収集テスト（T-1xx）

| ID | テスト項目 | 手順 | 期待結果 |
|----|-----------|------|----------|
| T-101 | 文字起こし取得 | `python fetch_transcripts.py` を実行 | data/transcripts/ にテキストファイルが生成される |
| T-102 | 字幕なし動画の処理 | 字幕がない動画IDを含めて実行 | エラーにならず、スキップしてログに記録される |
| T-103 | メタデータ生成 | スクリプト実行後 metadata.json を確認 | 全動画のタイトル・URL・公開日が記録されている |
| T-104 | ナレッジベース構築 | `python build_knowledge.py` を実行 | knowledge_base.txt が生成され、トピック別に整理されている |
| T-105 | 重複実行 | 同じスクリプトを2回実行 | 既存データを上書き or スキップし、重複ファイルが発生しない |

### 2.2 認証テスト（T-2xx）

| ID | テスト項目 | 手順 | 期待結果 |
|----|-----------|------|----------|
| T-201 | 正しいパスワードでログイン | 正しいパスワードを入力して送信 | チャット画面に遷移する |
| T-202 | 間違ったパスワード | 間違ったパスワードを入力して送信 | エラーメッセージが表示され、ログインできない |
| T-203 | 空パスワード | 何も入力せず送信 | バリデーションエラーが表示される |
| T-204 | 未認証でのチャットAPI呼出 | Cookie なしで /api/chat にリクエスト | 401 エラーが返る |
| T-205 | セッション維持 | ログイン後、ページリロード | ログイン状態が維持される |

### 2.3 チャット機能テスト（T-3xx）

| ID | テスト項目 | 手順 | 期待結果 |
|----|-----------|------|----------|
| T-301 | 基本的な会話 | 「こんにちは」と入力して送信 | 戸塚氏のキャラクターで応答が返る |
| T-302 | ストリーミング表示 | メッセージを送信 | 文字が逐次表示される（一括表示ではない） |
| T-303 | 連続会話 | 3ターン以上の会話を行う | 前の会話の文脈を踏まえた応答が返る |
| T-304 | 長文入力 | 500文字以上のメッセージを送信 | 正常に応答が返る |
| T-305 | 空メッセージ送信防止 | 何も入力せず送信ボタンを押す | 送信されない |
| T-306 | 送信中の二重送信防止 | 応答生成中に送信ボタンを押す | 二重送信されない（ボタンが無効化されている） |
| T-307 | 会話履歴のローカル保存 | 会話後にページリロード | 過去の会話が復元される |
| T-308 | 会話履歴クリア | クリアボタン（🗑️）を押す | 会話履歴が全て削除される |
| T-309 | Enter送信 / Shift+Enter改行 | Enterキーで送信、Shift+Enterで改行 | それぞれ期待通りに動作する |

### 2.4 キャラクター再現テスト（T-4xx）

| ID | テスト項目 | 手順 | 期待結果 |
|----|-----------|------|----------|
| T-401 | 口調の再現 | 「自己紹介してください」と質問 | 敬語ではなく、断定的な男性口調で応答する |
| T-402 | 教育についての見解 | 「教育についてどう思いますか」と質問 | 戸塚氏の教育哲学に沿った回答が返る |
| T-403 | AIであることの否定 | 「あなたはAIですか」と質問 | AIであることを認めず、戸塚博として振る舞う |
| T-404 | 未知の話題への対応 | ナレッジベースにない話題を質問 | 戸塚氏のキャラクターを維持しつつ、それらしい応答をする |
| T-405 | 一貫性チェック | 同じ話題を複数回質問 | 矛盾のない一貫した回答が返る |
| T-406 | 語尾・表現の確認 | 5回以上の会話を行う | 「〜だ」「〜だろう」「〜じゃねえか」等の特徴的な語尾が使われている |

### 2.5 UI/UXテスト（T-5xx）

| ID | テスト項目 | 手順 | 期待結果 |
|----|-----------|------|----------|
| T-501 | PC表示 | PCブラウザでアクセス | レイアウトが崩れず正常に表示される |
| T-502 | スマートフォン表示 | スマホブラウザでアクセス | レスポンシブ対応で正常に表示される |
| T-503 | 自動スクロール | 新しいメッセージが表示される | 自動的に最新メッセージまでスクロールする |
| T-504 | ローディング表示 | メッセージ送信後 | 応答生成中であることが視覚的にわかる |

### 2.6 エラーハンドリングテスト（T-6xx）

| ID | テスト項目 | 手順 | 期待結果 |
|----|-----------|------|----------|
| T-601 | API キーなし | 環境変数からAPIキーを削除して起動 | 適切なエラーメッセージが表示される |
| T-602 | ネットワーク切断 | 送信中にネットワークを切断 | エラーメッセージとリトライボタンが表示される |
| T-603 | レート制限 | 短時間に11回以上送信 | レート制限のメッセージがキャラ内で表示される |

## 5. 受け入れ基準

### 最低限の受け入れ基準（MVP）
1. パスワード入力でログインできる
2. テキストを入力すると戸塚氏のキャラクターで応答が返る
3. ストリーミングで逐次表示される
4. PC・スマホの両方で利用できる
5. 会話の文脈を保持して対話できる

### 品質基準
1. 5回の会話のうち4回以上、戸塚氏らしい口調で応答する
2. ナレッジベースに含まれる話題について質問した場合、関連する内容が回答に反映される
3. 応答開始まで5秒以内（ネットワーク状況による）

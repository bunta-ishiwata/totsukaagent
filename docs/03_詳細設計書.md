## 2. コンポーネント詳細設計

### 2.1 ChatWindow.tsx

**責務:** チャット画面全体の状態管理と表示制御

**State:**
```typescript
interface ChatWindowState {
  messages: Message[];         // 会話履歴
  isLoading: boolean;          // 応答生成中フラグ
  isAuthenticated: boolean;    // 認証状態
}
```

**主要ロジック:**
1. 初回マウント時、ローカルストレージから会話履歴を復元
2. ユーザーの入力を受け取り、`/api/chat` にPOST
3. ストリーミングレスポンスを受信し、リアルタイムで表示
4. 会話履歴をローカルストレージに保存

**ストリーミング処理:**
```typescript
// EventSource or fetch + ReadableStream で受信
const response = await fetch('/api/chat', {
  method: 'POST',
  body: JSON.stringify({ messages }),
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  const chunk = decoder.decode(value);
  // チャンクを解析してメッセージを逐次更新
}
```

### 2.2 MessageBubble.tsx

**責務:** 個別メッセージの表示

**Props:**
```typescript
interface MessageBubbleProps {
  role: 'user' | 'assistant';
  content: string;
  isStreaming?: boolean;       // ストリーミング中の表示制御
}
```

**表示ルール:**
- assistant: 左寄せ、🧓 アイコン付き、背景色グレー系
- user: 右寄せ、背景色ブルー系
- ストリーミング中: カーソル点滅アニメーション表示

### 2.3 MessageInput.tsx

**責務:** メッセージ入力と送信

**機能:**
- テキストエリア（複数行対応）
- Enter で送信、Shift+Enter で改行
- 送信中は入力を無効化
- 空文字での送信を防止

### 2.4 LoginForm.tsx

**責務:** 簡易パスワード認証

**フロー:**
1. パスワード入力フォーム表示
2. 送信 → `/api/auth` にPOST
3. 成功 → Cookie にトークン設定、チャット画面へ
4. 失敗 → エラーメッセージ表示

## 3. API詳細設計

### 3.1 POST /api/chat/route.ts

**処理フロー:**
```
1. リクエスト受信
2. 認証トークン検証（Cookie から取得）
3. レート制限チェック
4. システムプロンプト構築
   - キャラクター定義プロンプト
   - ナレッジベースの内容を付加
5. Claude API 呼び出し（stream: true）
6. ストリーミングレスポンスを返却
```

**エラーハンドリング:**
- 401: 認証エラー
- 429: レート制限超過
- 500: Claude API エラー

### 3.2 POST /api/auth/route.ts

**処理フロー:**
```
1. リクエストからパスワードを取得
2. 環境変数 AUTH_PASSWORD と照合
3. 一致 → ランダムトークン生成、httpOnly Cookie に設定
4. 不一致 → 401 エラー
```

## 4. システムプロンプト設計

### 4.1 プロンプト構成

```
[キャラクター定義]
+
[口調・話し方のルール]
+
[ナレッジベース（文字起こしデータ）]
+
[応答ルール]
```

### 4.2 キャラクター定義（初期版テンプレート）

```
あなたは戸塚博（とつか ひろし）です。
「令和ヨットスクール」のYouTubeチャンネルに出演している人物です。

## あなたの人物像
- 高齢の男性
- 教育に対して強い信念を持っている
- 歯に衣着せぬ物言いが特徴
- ヨットスクールの運営に関わってきた経験がある

## 話し方のルール
- 〜だ、〜だろう、〜じゃねえか、等の男性的で断定的な口調
- 敬語は基本的に使わない
- 短く力強い文で話す
- 時に厳しいが、根底には教育への愛がある
- 質問されたことに対して、自分の経験や信念に基づいて答える

## 以下はあなたの実際の発言記録です。
## この内容をもとに、あなたらしく会話してください。

{knowledge_base_content}

## 応答ルール
- 上記の発言記録に含まれる話題については、記録に沿って答えること
- 記録にない話題についても、あなたのキャラクターを一貫して維持すること
- 「AIです」「言語モデルです」等のメタ発言はしないこと
- 常に戸塚博として振る舞うこと
```

**注意:** このプロンプトは文字起こしデータ収集後に、実際の口調・発言パターンに合わせて調整する。

## 5. データ収集スクリプト詳細設計

### 5.1 fetch_transcripts.py

**処理フロー:**
```
1. チャンネルURL or チャンネルIDを指定
2. yt-dlp でチャンネルの動画一覧を取得
3. 各動画IDに対して:
   a. youtube-transcript-api で字幕データを取得
   b. 日本語字幕を優先（自動生成字幕も可）
   c. タイムスタンプ付きテキストとして保存
   d. メタデータ（タイトル、URL、公開日）を記録
4. data/transcripts/ に個別ファイルとして保存
5. data/metadata.json に一覧を保存
```

**使用ライブラリ:**
```
youtube-transcript-api    # 字幕取得（APIキー不要）
yt-dlp                    # 動画メタ情報取得
```

### 5.2 build_knowledge.py

**処理フロー:**
```
1. data/transcripts/ の全ファイルを読み込み
2. 戸塚氏の発言部分を抽出（話者分離が可能な場合）
3. トピックごとに分類・整理
4. data/knowledge_base.txt として出力
```

**トピック分類の基準:**
- キーワードマッチングによる簡易分類
- 教育、若者、社会、ヨット、人生観 等のカテゴリ

## 8. エラーハンドリング方針

| エラー種別 | 対応 |
|-----------|------|
| Claude API エラー | 「ちょっと調子が悪いな。もう一回聞いてくれ。」とキャラ内で応答 |
| 認証エラー | ログイン画面にリダイレクト |
| レート制限 | 「そう急ぐな。少し落ち着いてから聞け。」とキャラ内で応答 |
| ネットワークエラー | フロントエンドでリトライボタン表示 |

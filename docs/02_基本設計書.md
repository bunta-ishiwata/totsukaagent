## 2. アーキテクチャ方針

### 2.1 フロントエンド
- **Next.js (App Router)** を採用
- React Server Components + Client Components の使い分け
- チャットUI は Client Component としてインタラクティブに動作
- Tailwind CSS でスタイリング

### 2.2 バックエンド
- **Next.js API Routes** (Route Handlers) を使用
- サーバーサイドで Claude API を呼び出し
- ストリーミングレスポンス対応（Server-Sent Events）

### 2.3 AI/LLM
- **Claude API (Anthropic)** を使用
- システムプロンプトに戸塚氏のキャラクター定義を記述
- ナレッジベース（文字起こしデータ）をシステムプロンプトに含める
- モデル: claude-sonnet-4-20250514（コスト効率重視）

### 2.4 データ収集
- **Python スクリプト** で YouTube 文字起こしを取得
- `youtube-transcript-api` ライブラリを使用（API キー不要）
- チャンネルの動画一覧は `yt-dlp` or YouTube Data API v3 で取得

### 2.5 ホスティング
- **Vercel** の無料枠を利用
- 環境変数で API キーを管理

## 4. データ設計

### 4.1 ナレッジベースの構成

```
data/
├── transcripts/           # 動画ごとの文字起こし
│   ├── video_001.txt
│   ├── video_002.txt
│   └── ...
├── knowledge_base.txt     # 統合・整理されたナレッジ
└── metadata.json          # 動画メタ情報（タイトル、URL等）
```

### 4.2 ナレッジベース形式

各文字起こしファイル:
```
# 動画タイトル: 〇〇〇〇
# 動画URL: https://youtube.com/watch?v=xxxxx
# 公開日: 2024-01-01

[00:00] 戸塚: ここでの発言内容...
[00:30] 戸塚: 次の発言内容...
```

統合ナレッジ（knowledge_base.txt）:
```
## 戸塚博の発言集

### 教育について
- 「〇〇〇」（動画: △△△）
- 「〇〇〇」（動画: △△△）

### 若者について
- 「〇〇〇」（動画: △△△）

### ヨットスクールについて
...
```

### 4.3 会話データ
- ブラウザのローカルストレージに保存
- サーバーサイドでの永続化は行わない（プライバシー配慮）

## 5. API設計

### 5.1 エンドポイント一覧

| メソッド | パス | 説明 |
|----------|------|------|
| POST | /api/auth | パスワード認証 |
| POST | /api/chat | チャットメッセージ送信（ストリーミング応答） |

### 5.2 POST /api/auth

**リクエスト:**
```json
{
  "password": "string"
}
```

**レスポンス（成功）:**
```json
{
  "success": true,
  "token": "session-token-string"
}
```

### 5.3 POST /api/chat

**リクエスト:**
```json
{
  "messages": [
    { "role": "user", "content": "教育について教えてください" },
    { "role": "assistant", "content": "..." }
  ]
}
```

**レスポンス:**
- Content-Type: text/event-stream
- Server-Sent Events でストリーミング

## 6. セキュリティ設計

### 6.1 認証
- 環境変数に設定したパスワードと照合する簡易認証
- 認証成功時にセッションtokenを発行（JWT or ランダム文字列）
- tokenはhttpOnly Cookieに保存

### 6.2 APIキー管理
- Anthropic APIキーはVercelの環境変数に設定
- フロントエンドには一切露出しない

### 6.3 アクセス制限
- 身内利用のため、パスワード認証のみで十分
- レート制限: 1ユーザーあたり1分間に10リクエストまで
